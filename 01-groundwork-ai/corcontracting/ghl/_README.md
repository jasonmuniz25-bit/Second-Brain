# ghl — GoHighLevel build documentation

Document each GHL workflow, pipeline, and automation as a separate note so you can hand the system off to a future operator (or to Claude for analysis) without explaining live state from memory.

Suggested notes per workflow:
- `wf-[name].md` — trigger, steps, exit conditions, edge cases, last-modified date

Use `templates/ghl-workflow.md` for new workflow docs.
