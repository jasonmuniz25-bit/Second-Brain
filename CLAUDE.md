# CLAUDE.md

> Read this file first. It tells you what this vault is, how to navigate it, and how to work with Jay.

## what this vault is

Jay's Second Brain. Personal knowledge base + working space for **GroundWork AI** (his lead-gen business for home-services contractors), **CorContracting** (his brother Corey's NJ remodeling company — first GroundWork client, live system), an HVAC sales coaching project, and personal life (fitness, BJJ, the long-game).

## start every session with these three reads

1. `maps/me.md` — who Jay is, how he thinks, how to work with him. **Required.**
2. `AI/sessions/` — open the most recent file. That's where the last session left off.
3. `00-inbox/` — anything in there is unprocessed. Mention it if relevant.

After you read those, state in one sentence what context you found, then ask what we're working on.

## navigation

The folder map lives in `maps/vault-map.md`. Every folder also has a `_README.md` explaining what belongs in it. Trust those, not your memory.

## skills

`maps/skills/` contains the trigger phrases and formats for the workflows Jay uses repeatedly. Read the file when its trigger fires — don't improvise the format.

- `session-summary.md` — fires on "wrap this up," "close this out," "session summary"

## file rules

- Lowercase kebab-case filenames. No spaces.
- Session summaries: `AI/sessions/YYYY-MM-DD-topic.md`
- Daily notes: `daily-notes/YYYY-MM-DD.md`
- Never overwrite existing files — always create new ones (use `-v2.md` if updating)
- AI-generated files get `ai-generated: true` in frontmatter

## hard rules — don't break these

1. **Do not change live GHL workflows or Airtable automations** without explicit instruction. CorContracting is a real client running real money.
2. **Do not send anything to Corey or any external person** without explicit confirmation from Jay.
3. **Do not suggest Jay quit his HVAC job.** He knows. Not the question.
4. When Jay is overthinking pace or timeline, **don't validate the spiral — redirect to the next concrete action.**
5. Check `AI/sessions/` and `maps/me.md` before asking Jay to re-explain context that's already documented.
6. **Propose updates to `maps/me.md` — never overwrite it without Jay's explicit confirmation.**

## tone (full version in maps/me.md)

Direct. No filler. Short paragraphs over bullets unless structure helps. Push back on pitches; just answer learning questions. Don't treat Jay's agreement with a suggestion as a decision — he'll explicitly lock things in.
